Frontend cannot connect to MongoDB directly. Browsers don't support MongoDB's wire protocol. You need a backend server (Node.js + Express + Mongoose) between your frontend and MongoDB. Below is a complete guide with code.

How to Connect to MongoDB (Step by Step)

  1. Set up MongoDB Atlas (Cloud) or Local MongoDB

    Create a free cluster at mongodb.com/cloud/atlas or install MongoDB locally. Get your connection string like: mongodb+srv://user:pass@cluster.mongodb.net/voyago

  2. Create a Node.js + Express Backend

    This backend will expose REST API endpoints that your frontend calls with fetch().

  3. Define Mongoose Models

    Create schemas for Vehicles, Packages, and Bookings that map to MongoDB collections.

  4. Build CRUD API Endpoints

    GET, POST, PUT, DELETE routes for each resource. Add JWT auth for security.

  5. Update This Frontend to Call the API

    Replace the in-memory DB object calls with fetch('http://localhost:3000/api/vehicles') etc.

Backend Server Code (server.js)

Save this as server.js and run npm init -y && npm install express mongoose cors dotenv then node server.js

// server.js — Backend API for Voyago const express = require('express'); const mongoose = require('mongoose'); const cors = require('cors'); const app = express(); app.use(cors()); app.use(express.json()); // Connect to MongoDB mongoose.connect('mongodb+srv://user:pass@cluster.mongodb.net/voyago') .then(() => console.log('MongoDB connected')) .catch(err => console.error(err)); // Vehicle Schema const vehicleSchema = new mongoose.Schema({ name: String, type: String, brand: String, year: Number, seats: Number, fuel: String, transmission: String, pricePerDay: Number, status: { type: String, default: 'active' }, images: [String], features: [String], routes: [{ name: String, emoji: String }], createdAt: { type: Date, default: Date.now } }); const Vehicle = mongoose.model('Vehicle', vehicleSchema); // CRUD Endpoints app.get('/api/vehicles', async (req, res) => { const vehicles = await Vehicle.find(); res.json(vehicles); }); app.post('/api/vehicles', async (req, res) => { const vehicle = new Vehicle(req.body); await vehicle.save(); res.status(201).json(vehicle); }); app.put('/api/vehicles/:id', async (req, res) => { const vehicle = await Vehicle.findByIdAndUpdate( req.params.id, req.body, { new: true } ); res.json(vehicle); }); app.delete('/api/vehicles/:id', async (req, res) => { await Vehicle.findByIdAndDelete(req.params.id); res.json({ success: true }); }); app.listen(3000, () => console.log('API running on port 3000'));

// Replace: DB.vehicles.update(id, data) const res = await fetch(`http://localhost:3000/api/vehicles/${id}`, { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(updatedData) });

// Replace: DB.vehicles.remove(id) await fetch(`http://localhost:3000/api/vehicles/${id}`, { method: 'DELETE' });

Why not MongoDB Realm / Stitch? MongoDB does offer a client-side SDK (Realm), but it requires a MongoDB Atlas App Services backend configuration, has its own auth system, and still isn't a "direct connection" from pure frontend code. A proper Express API is simpler, more standard, and gives you full control.
